## Overview

예약 시스템의 Data-Model, Entity에 대한 설계.
예약 시스템의 핵심 시나리오 예약 생성, 예약 취소를 동작시키기 위한 핵심 Data Model, Domain Entity 두가지 축에 대해 설계한 내용

- reservation_slots 에서 capacity, remaining이 존재함 = 하나의 예약 대상에 대해 여러명이 신청할 수 있는 구조, 대상에 대해 좌석 구분은 존재하지 않는다.

## Data-Model

Table:

users
reservable_resources
reservation_slots
reservations

### 1. Users

id: uuid primary key
name: string
email: string unique key
created_at: Date
updated_at: Date

### 2. Reservable_Resources

id: uuid primary key
title: string
description: string
status: 예약가능, 예약불가능, 폐강
created_at: Date
updated_at: Date

### 3. Reservation_Slots

id: uuid primary key
resource_id: uuid foreign key
start_at: Date
end_at: Date
capacity: number
remaining: number
version: number
created_at: Date
updated_at: Date

### 4. Reservations

id: uuid primary key
user_id: uuid foreign key
slot_id: uuid foreign key
status: PENDING | CONFIRM | CANCEL | NOSHOW | COMPLETE
created_at: Date
updated_at: Date

### 5. Reservation_Status_Histories

id: uuid primary key
reservation_id: uuid foreign key
reason: string
before_status: PENDING | CONFIRM | CANCEL | NOSHOW | COMPLETE
after_status: PENDING | CONFIRM | CANCEL | NOSHOW | COMPLETE
created_at: Date


## Domain Entity

### 1 ReservationStatus 

- enum = [PENDING, CONFIRM, CANCEL, NOSHOW, COMPLETE, EXPIRED]

### 2. ReservaionSlots

- reserveOne()
- releaseOne()
- haveRemaining()
- isExpired()


### 3. Reservations

강의 좌석 1건에 대한 실제 예약 정보. 강의 좌석간에 차이는 없음

- create(): status: PENDING
- markConfirm(): status: CONFIRM
- cancelByUser(): status: CANCEL
- cancelByAdmin(): status: CANCEL
- markNoShow(): status: NOSHOW
- complete(): status: COMPLETE
- markExpired(): status: EXPIRED
